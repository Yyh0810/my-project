# Templates

This directory is reserved for scalable template material used by the orchestrator.

Use it for:

- article skeletons by scenario or article type
- title strategy packs
- review rubric variants
- visual brief presets
- prompt packs
- reusable component snippets
- JSON/YAML schema examples

Suggested future layout:

```text
templates/
  articles/
  titles/
  reviews/
  visuals/
  prompts/
  components/
  schemas/
```

Keep executable orchestration logic in the sibling capability directories. Templates should be data or text artifacts that scripts can load.
