# 00 运行契约

本文件是所有阶段的硬约束。涉及文章决策、文件修改、上传、草稿箱、发布、凭证、远程服务调用时，先遵守本文件。

## 禁止事项

| 禁止项 | 具体要求 |
| --- | --- |
| 自作主张 | 不擅自决定账号、品牌定位、正式发布、群发、上传、远程生图、跳过审稿 |
| 跳步骤 | 缺正文不排版，缺预览不进草稿箱，缺二次确认不群发 |
| 伪完成 | 没有文件、预览、检查结论或接口回执，不说“已完成” |
| 覆盖源稿 | 默认写新文件或临时稿，不覆盖原始 Markdown 和用户供图 |
| 密钥风险 | 不索要、不打印、不复制 API key、AppSecret、access token |
| 状态混淆 | 预览不等于草稿箱，草稿箱不等于正式发布，图片方案不等于图片文件 |
| 静默降级 | 工具不可用或能力降低时，说明差异后再继续 |
| 编造事实 | 漏洞、政策、产品版本、客户案例、指标等不确定时查证或标注待确认 |
| 擅自调用外部能力 | 用户未明确要求时，不调用其他 skill、`md2wechat` 或其他外部能力；未经明确要求也不调用 image-2 / 当前 Codex 图像工具生图 |

## 质量门禁

每个维度按 0-5 分评分：

- 5：可直接进入下一阶段。
- 4：有小问题，不阻塞。
- 3：勉强可用，建议修改。
- 2：明显不足，必须修改。
- 1：严重不足，暂停推进。
- 0：缺失或不可用。

默认通过线：

| 进入阶段 | 通过线 |
| --- | --- |
| 进入排版 | 内容质量、可信度、结构完整度均不低于 4 |
| 进入草稿箱 | 内容、排版、图片、元数据、发布安全均不低于 4 |
| 正式发布 | 全部核心维度不低于 4，且无红线问题 |

红线问题：

- 标题承诺正文无法兑现。
- 关键事实未经确认却写成确定结论。
- 存在未处理的敏感信息。
- 预览不可用或图片不显示。
- 封面缺失却要进草稿箱。
- 草稿箱没有回执却声称成功。

## 副作用边界

以下动作必须获得用户明确意图：

- 上传图片。
- 创建或更新草稿。
- 调用 image-2 / 当前 Codex 图像工具或其他远程图片生成服务。
- 调用微信接口。
- 正式发布或群发。

调用 image-2 / 当前 Codex 图像工具前必须先确认用户明确要求，并明确：图片用途、画幅、主题、风格、是否含文字、敏感信息边界。调用后必须给出图片文件状态和质量检查，不得把“提示词已写好”说成“图片已生成”。

微信接口脚本必须默认 dry-run。只有同时满足以下条件，才允许带 `--execute` 执行：

- 用户明确授权本次上传、草稿箱或素材动作。
- 已说明动作会产生的副作用。
- 目标公众号账号明确。
- 本地环境变量已配置，且不在聊天中展示 AppSecret、access token。
- 当前稿件通过对应阶段质量门禁。

这些表达不得当作正式发布确认：`可以了`、`发一下`、`放公众号`、`走吧`、`你看着办`、`按默认来`。它们最多表示可以继续做草稿箱准备。

正式发布需要两次确认：

1. 用户明确要求正式发布或群发。
2. 终审后，再确认文章标题、账号和发布边界。

## 质量检查输出

```text
质量结论：通过 / 需修改 / 暂停推进
内容质量：x/5
可信度：x/5
结构完整度：x/5
排版质量：x/5
图片质量：x/5
草稿箱就绪：x/5
红线问题：
主要问题：
建议修改：
下一步：
```

不适用的维度写“未进入该阶段”，不要随意打满分。
