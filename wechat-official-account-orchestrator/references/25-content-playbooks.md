# 25 内容生产打法

用于从 0 生成公众号文章选题、标题、大纲和改稿方向。本文只提供结构化打法，不替代事实核验。

## 通用原则

- 先确定文章类型，再生成标题和大纲。
- 所有缺失事实写成 `待核验`，不得补造客户、数字、漏洞编号、产品版本或指标。
- 标题不能承诺正文无法证明的结论。
- 产品连接强度分为 `none`、`light`、`medium`、`strong`；默认 `light`。
- 写作顺序建议为：资料台账 -> 论证链 -> playbook -> 标题策略 -> 大纲 -> 成稿写作包 -> 初稿/改稿 -> 审稿质检 -> 组件编排 -> 排版预览 -> 草稿箱。

## 成稿写作包

大纲只回答“写什么顺序”，不能直接保证初稿质量。进入模型写作前，应先生成成稿写作包：

```bash
python scripts/content/article_argument_chain.py --topic "AI Agent 安全运营" --type security-analysis --ledger article.research-ledger.json --out article.argument-chain.json
python scripts/content/article_writing_pack.py --topic "AI Agent 安全运营" --type security-analysis --audience "安全运营团队" --outline article.outline.md --argument-chain article.argument-chain.json --out article.writing-pack.json
```

写作包用于约束模型和人工协作，必须包含：

- 核心角度：文章要解释或证明的主判断。
- 读者任务：读者看完后能判断什么、行动什么。
- 事实台账：哪些事实必须核验，来源在哪里。
- 分段任务：每节的写作目标、必须包含、避免事项。
- 模型提示词：把模型写作限定在已提供事实和待核验标记内。
- 人工复核点：标题兑现、事实边界、产品连接、授权素材。

没有材料时，写作包只能生成可控初稿提示，不能把 `待核验` 写成确定事实。

## 文章类型

| 类型 | 中文名 | 适用场景 |
| --- | --- | --- |
| `security-analysis` | 安全分析 | 漏洞、攻击面、风险链路、防护策略 |
| `agent-observation` | AI Agent 观察 | Agent 使用变化、工程实践、工具链趋势 |
| `product-launch` | 产品发布 | 新能力、版本更新、解决方案说明 |
| `industry-observation` | 行业观察 | 市场现象、技术趋势、监管变化 |
| `case-review` | 案例复盘 | 项目复盘、事件复盘、运营复盘 |

## 安全分析

- 标题方向：风险链路、盲区、检查清单、处置优先级。
- 标准结构：背景 -> 影响对象 -> 风险链路 -> 检查建议 -> 产品连接/行动。
- 开头方式：从具体风险或运营问题进入。
- 证据要求：漏洞编号、影响范围、利用状态、日期必须有来源。
- CTA 风格：检查资产、复核暴露面、补齐监测和处置动作。
- 禁用表达：全网沦陷、必炸、零风险、彻底解决、攻击复现步骤。

## AI Agent 观察

- 标题方向：工作方式变化、协作边界、工程约束、真实收益。
- 标准结构：现象 -> 为什么发生 -> 对团队影响 -> 落地边界 -> 下一步。
- 开头方式：从一个真实工作流变化或协作问题进入。
- 证据要求：工具能力、案例、效率提升必须标注来源或待核验。
- CTA 风格：选择一个低风险场景试点，明确评估指标。
- 禁用表达：Agent 替代所有人、完全自动化、无需人工审核。

## 产品发布

- 标题方向：能力上线、场景解决、版本价值、使用路径。
- 标准结构：场景问题 -> 新能力 -> 使用路径 -> 能力边界 -> 下一步。
- 开头方式：从用户场景和已有痛点进入，不直接堆功能。
- 证据要求：版本号、能力范围、客户效果、指标必须可核验。
- CTA 风格：试用、预约演示、查看文档、联系团队。
- 禁用表达：行业第一、唯一、秒杀、100% 提升、永久解决。

## 行业观察

- 标题方向：趋势判断、变化原因、对团队影响、决策建议。
- 标准结构：现象 -> 证据 -> 判断 -> 边界 -> 行动建议。
- 开头方式：从一个可观察现象进入，避免空泛趋势词。
- 证据要求：报告、调研、排名、市场规模必须有来源。
- CTA 风格：给出判断标准或内部讨论问题。
- 禁用表达：颠覆一切、时代变了、所有公司都在做。

## 案例复盘

- 标题方向：复盘对象、关键转折、经验教训、改进清单。
- 标准结构：背景 -> 目标 -> 过程 -> 问题 -> 经验 -> 后续动作。
- 开头方式：从复盘目标和约束进入。
- 证据要求：客户、项目、结果、截图、数据必须授权或脱敏。
- CTA 风格：复用清单、避免同类问题、推动下一轮改进。
- 禁用表达：未经授权客户名称、合同信息、报价、内部策略。
