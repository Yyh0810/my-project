# 70 正式发布门禁

用于草稿箱之后的正式发布/群发路径。正式发布不是默认动作，只有用户明确要求并二次确认后才进入本阶段。

## 阶段位置

```text
选题 -> 写稿 -> 审稿 -> 排版 -> 图片 -> 草稿箱 -> 后台人工终审 -> 正式发布门禁 -> 正式发布/群发
```

默认通路停在草稿箱。若用户只说“发一下”“走吧”“可以了”，最多理解为继续做草稿箱准备，不进入正式发布。

## 进入条件

必须全部满足：

- 草稿箱已创建，且微信公众号后台可见。
- 后台人工终审完成。
- 最终标题、摘要、封面、正文图片均已确认。
- 质量结论为通过。
- 目标公众号账号明确。
- 用户明确要求“正式发布/群发/发出去”。
- 在看到最终标题、账号和发布边界后，用户给出第二次明确确认。

缺任一项，停止在草稿箱或就绪报告。

## 门禁检查文件

正式发布前创建一个确认 JSON，例如 `article.release.json`：

```json
{
  "account_name": "目标公众号名称",
  "article_title": "文章标题",
  "draft_status": "草稿箱已创建",
  "backend_review": true,
  "quality_result": "通过",
  "second_confirmation": true,
  "release_boundary": "正式发布到目标公众号，不自动转载到其他渠道",
  "scheduled_time": "",
  "timezone": "Asia/Shanghai"
}
```

运行门禁检查：

```bash
python scripts/wechat/formal_release_gate.py article.release.json --out article.release.gate.json
```

`ok: true` 只表示门禁材料齐全，不表示已经发布。

## 执行方式

首版建议采用微信公众号后台人工点击发布/群发：

- 操作人登录微信公众号后台。
- 打开草稿箱对应文章。
- 核对标题、摘要、封面、正文、图片、原创/转载、评论设置。
- 根据确认的发布边界点击正式发布或群发。
- 保存后台发布结果截图或回执信息。

如果未来接入微信正式发布 API，必须先单独验证官方接口、账号权限、发布配额和回执字段，再加入脚本；不得把草稿箱 API 成功误认为正式发布成功。

## 发布后回执

正式发布后记录：

```text
账号：
文章标题：
发布时间：
操作人：
发布方式：后台人工 / API
后台状态：
文章链接：
回执截图或记录：
异常：
```

## 禁止事项

- 不用“草稿箱成功”代替“正式发布成功”。
- 不在没有二次确认时发布。
- 不跨账号发布。
- 不跳过后台最终人工检查。
- 不发布含待确认事实、坏图、错误封面或错账号的文章。
