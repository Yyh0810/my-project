# 45 小创意组件素材库

用于公众号稿件中的局部视觉点缀。组件来自公开公众号排版工具常见模式的抽象，不复制第三方成品样式。

## 设计原则

- 先服务阅读节奏，再考虑装饰。
- 一个模块只承担一个功能：转场、强调、旁白、清单、步骤、数字、对比或行动。
- 单篇 1500-2500 字文章默认使用 2-4 个模块。
- 不连续堆叠两个以上模块。
- 科技、安全、产品类稿件要克制，避免娱乐化。
- 政务、客户汇报、正式稿优先用 `quote`、`insight`、`checklist`，慎用 `agentbox`。

## 模块清单

| 模块 | 功能 | 适用位置 | 注意 |
| --- | --- | --- | --- |
| `divider` | 短分割线/转场标签 | 大段落切换、章节缓冲 | 不承载核心观点 |
| `question` | 引出核心疑问 | 章节开头、冲突点前 | 保持短句 |
| `quote` | 强调关键判断 | 观点句、引用句、金句 | 不放长篇解释 |
| `insight` | 核心洞察卡 | 小结、判断、策略建议 | 适合正式稿 |
| `agentbox` | 标签信息卡 | AI/Agent 视角、产品旁白、案例插话 | 不替代正文 |
| `checklist` | 检查清单 | 发布前检查、应急步骤、能力要点 | 每项尽量一行 |
| `steps` | 步骤/时间线 | 流程、路径、处置顺序 | 控制在 3-6 步 |
| `stat` | 关键数字 | 数据点、风险规模、效率指标 | 数字必须可核验 |
| `compare` | 左右对比 | 旧做法 vs 新做法、误区 vs 建议 | 两侧文字长度接近 |
| `cta` | 行动提示 | 文末、下载/咨询/下一步 | 不写夸张营销话术 |
| `warning` | 风险提示 | 漏洞风险、发布阻断、注意事项 | 不制造恐慌 |
| `definition` | 概念解释 | 术语定义、产品概念、技术背景 | 先解释边界，再展开 |
| `summary` | 本节小结 | 章节收束、关键结论复盘 | 控制 3-5 条 |
| `source` | 依据与说明 | 数据来源、事实边界、口径说明 | 不替代正式引用 |

## Markdown 写法

```markdown
:::divider
阶段小结
:::

:::question
为什么这次不是“多扫几个漏洞”就能解决？
:::

:::quote
真正的风险，不在于漏洞数量，而在于**攻击路径已经形成闭环**。
:::

:::insight 核心洞察
安全运营的问题，往往不是缺工具，而是缺少能把资产、漏洞、权限和行为串起来的判断链路。
:::

:::agentbox AgentBox
如果让 Agent 接手这类检查，它不应该只输出漏洞列表，而要解释：为什么这些记录共同指向一条可利用路径。
:::

:::checklist 发布前确认
- 标题承诺正文已经兑现
- 图片都能显示
- 关键事实已有来源
:::

:::steps 应急处置路径
- 确认影响资产
- 复核暴露面
- 检查利用痕迹
- 输出修复优先级
:::

:::stat 82 个
被集中利用的漏洞数量，需要结合暴露资产和攻击路径一起看。
:::

:::compare 两种排查思路
只看 CVE 列表，按严重等级排序。
---
结合资产、权限、日志和暴露面，判断真实攻击路径。
:::

:::cta 下一步
先完成本地预览和素材清单检查，再决定是否进入公众号草稿箱。
:::

:::warning 风险提示
如果 HTML 里仍然引用本地图片路径，草稿箱中可能无法显示正文图片。
:::

:::definition 判断链路
把资产、漏洞、权限、日志和处置动作串起来的分析路径，而不是孤立罗列信息。
:::

:::summary 本节小结
- 标题要兑现正文承诺
- 图片状态要明确
- 草稿箱不等于正式群发
:::

:::source 依据与说明
本文中的接口状态以本地脚本输出和微信公众号后台回执为准。
:::
```

## 使用建议

- 开头不要过早堆组件，先让读者进入主题。
- 中段用 `question` 或 `divider` 做节奏切换。
- 重要判断用 `quote` 或 `insight`，不要两者连续出现。
- 技术流程用 `steps`，核对项用 `checklist`。
- 风险阻断用 `warning`，不要用夸张标题放大焦虑。
- 术语第一次出现时可用 `definition`，不要把正文拆成词典。
- 章节结束可用 `summary`，但不要每一节都机械添加。
- 数据、案例或接口状态需要边界说明时用 `source`。
- 数据必须可靠，不能为了视觉效果编造 `stat`。
- 文末 CTA 要具体、克制，避免“速来”“爆款”等强营销词。

## 质量检查

使用小创意模块后检查：

- 是否提升理解，而不是增加噪音。
- 是否与当前预设风格一致。
- 是否在手机宽度下不超过两屏连续装饰。
- 是否存在未经核验的数字、案例或客户暗示。
- 是否有多余的拟人化、娱乐化表达。
